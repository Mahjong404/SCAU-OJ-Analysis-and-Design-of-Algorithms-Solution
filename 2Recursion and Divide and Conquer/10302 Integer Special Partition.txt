//10302 整数的特殊划分
#include <iostream>
using namespace std;

const int MOD = 1000000000;
int dp[10001];

int main() {
    int n;
    cin >> n;

    dp[0] = 1;
    // 完全背包：每个2的幂次无限使用
    for (int p = 1; p <= n; p <<= 1)
        for (int i = p; i <= n; i++)
            dp[i] = (dp[i] + dp[i - p]) % MOD;

    cout << dp[n];

    return 0;
}

Description
★问题描述：
给定一个正整数N，将其分解为若干个整数的和，且这些整数都是2的 k 次方(k>=0)，
请问共有多少种分解方法？
例如，对于整数5，有
5=1+1+1+1+1；
5=1+1+1+2；
5=1+2+2；
5=1+4
共4种分解方法。
对于整数8，有
8=1+1+1+1+1+1+1+1；
8=1+1+1+1+1+1+2；
8=1+1+1+1+2+2；
8=1+1+2+2+2;
8=2+2+2+2;
8=1+1+1+1+4;
8=1+1+2+4;
8=2+2+4;
8=4+4;
8=8
共10种分解方法。

输入格式
每组输入数据仅包含一个整数 N (1 <= N <= 10000)。
注意:这里原来题目是1 <= N <= 100000,现在改为:1 <= N <= 10000

输出格式
输出一个整数M。M为分解方法数，由于M可能很大，请输出 M%1000000000。

输入样例
5

输出样例

提示
定义如下：
n：正整数
2^i：定义为"特别加数"(i>=0)，特别加数为1 ，2 ，4 ，8 ，16 ，32 ，64 ，128，…
2^k：定义为"最大的特别加数"，k=取整(logn)

先计算加数<=1 的情况（即最大的特别加数=1），问题变为计算n-1这个数且最大的特别加数=1的分解个数；
再计算加数<=2 的情况（即最大的特别加数=2），问题变为计算n-2这个数且最大的特别加数=2分解个数；

……
将以上分解个数合计即可。
设f(n, k)：表示对n进行特殊加数的划分，特殊加数均<=2^k（即最大的特殊加数为2^k）的划分个数，这里n>=1，0<=k<=logn。

f(n, 0) = 1;
f(n, k) = f(n, 取下整（logn）)， k>=取下整(logn)
f(n, k) = sum of f(n-2^i, i) | for i=0 to i=k        k>0
注意：
1）这个问题组合数量庞大，如果直接用递归至顶向下的计算实现，将产生大量重复计算而使运行时间超时。
2）应设法将已经计算过的解记录下来，而避免重复计算。
