/*
Description
也是最优求幂的问题：给定一个正整数n和一个实数x，如何用最少的乘法次数计算出x^n。
（注：此OJ系统不方便输入有上下标的公式，所以如下都用x^n来表示x的n次幂）

比如，可用6次乘法逐步计算x^23，这样：x, x^2, x^3, x^5, x^10, x^20, x^23。也可
以是：x, x^2, x^4, x^5, x^9, x^18, x^23。

那计算x^23的幂序列中各幂次：1, 2, 3, 5, 10, 20, 23 就组成了一个关于整数23的加
法链；1, 2, 4, 5, 9, 18, 23 也组成一个加法链，这
两个加法链都是最短的，因为长度为6，因为可以证明，x^23至少需要6次乘法。

在一般情况下，计算x^n的幂序列中各幂次组成正整数n的一个加法链：

  1 = a0 < a1 < a2 < ... < ar = n     ai = aj+ak, k<=j<i, i=1,2,...,r

所以，上述最优求幂问题等价于正整数n的最短加法链问题，即，求n的一个加法链使其长度
达到最小。

输入格式
输入给定正整数n （n<=1000）。

输出格式
第一行：计算正整数n的加法链的最短的长度，（这个长度不会超过100）
第二行：其最短加法链本身。

当最短加法链并不止一种时，仅优先输出链靠前的数越大的这种。
比如：题上所列23的两种最短加法链：
1, 2, 3, 5, 10, 20, 23；
1, 2, 4, 5, 9, 18, 23。

前两个都一样1,2，第三个数有3或4，优先输出4的这条加法链。

输入样例
23

输出样例
6
1 2 4 5 9 18 23

提示
这个问题的状态空间树不是学习过的规则形状的子集树或排列树，而是第一层1个儿子，
第二层2个儿子，……，第n层n个儿子的树。

形状如下图所示：
<left><img src="pic/17093_1417748202823_31737.jpg"></left>

你可能要多次搜索如上这种多层的树，直到找到为止，搜索过程结束。
第一次搜索2层的树；
第二次搜索3层的树；
第三次搜索4层的树；
……
直到找到为止。

这个树的特点如下：
若树高为step层，首层的元素1为第一层，用a[1]...a[step]来记录从根到当前
结点的路径，代表已试探的加法链序列，初始化：a[1]=1。
（1）当前结点为第step层，a[1]...a[step]都已经试探好确定下来了，则下一
层（step+1层）的结点元素为：
a[step]+a[step], a[step]+a[step-1],...,a[step]+a[1]。
共step个儿子结点。
（2）当前结点的a[step]==n时，找到解了，可以退出搜索的过程。
