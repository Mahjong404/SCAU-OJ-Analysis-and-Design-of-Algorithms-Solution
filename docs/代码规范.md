# 代码规范

> 本项目 C++ 代码的编写约定。面向 SCAU OJ 提交，简洁优先。
> 风格参考 `Programming and Algorithm Contest` 项目。

## 目录结构

```
{章节目录}/
├── {题号} {英文题目名}/
│   ├── main.cpp              ← 唯一源文件
│   └── problem.png           ← 仅题目描述为图片时保留
├── {题号} {中文题目名}.txt    ← 与 main.cpp 内容相同，放在章节级别
├── {题号} {英文题目名}/
│   └── main.cpp
├── {题号} {中文题目名}.txt
└── ...
```

- 每个题目一个独立文件夹，命名 `{题号} {英文题目名}/`，内仅含 `main.cpp`
- 每个题目对应一个 `.txt` 文件，命名 `{题号} {中文题目名}.txt`，**放在章节级别**（与文件夹并列），内容与 `main.cpp` 完全一致
- 题目描述为图片时：将图片保存为 `problem.png` 放入文件夹，并在 `main.cpp` 末尾注释中还原文字描述

### 首行注释

每个 `main.cpp` 和 `.txt` 的第一行必须是：

```
//{题号} {中文题目名}
```

示例：`//9715 相邻最大矩形面积`

**不要**写作者/日期头部注释。

## 头文件

**仅使用以下头文件**：

```cpp
#include <iostream>
#include <algorithm>
#include <vector>
#include <string>
#include <set>
#include <stack>
#include <queue>
#include <array>
#include <map>
#include <cctype>
#include <cstdio>
#include <cstdlib>
#include <utility>
#include <iomanip>

using namespace std;
```

- 按需引入，不堆砌；`using namespace std;` 紧跟 include 之后

## main.cpp 结构

```cpp
#include <iostream>
using namespace std;

int main() {
    // 代码
    return 0;
}

/*
Description
题目描述……
输入格式
……
输出格式
……
输入样例
……
输出样例
……
提示
……
*/
```

- **不写**文件头部作者/日期注释（与参考项目一致）
- 代码在上，题目描述以 `/* */` 块注释放在文件末尾
- 描述块包含：Description、输入格式、输出格式、输入/输出样例、提示

## 代码风格

- `int main()` 声明写在一行，开括号不换行
- 紧凑风格：`cin>>n;` 无空格，简单 if/else 可同行
- 辅助函数写在 `main()` 前
- 不做多余抽象，不写类，一个 `main()` 解决问题
- 竞赛风格：不做防御性编程，不做输入合法性检查

## I/O

- 用 `cin`/`cout`，不用 `scanf`/`printf`
- 浮点保留小数位用 `<iomanip>` 的 `fixed` + `setprecision()`

## 注释

- 中文简注，只写关键思路
- 不逐行翻译代码

## 变量命名

- 简洁优先：`n`, `a`, `i`, `j` 足够
- 有语义需要时用英文，如 `maxLen`, `numerator`

## 描述完整性校验

写完 `main.cpp` 后，必须**逐题对比 OJ 原文**确认描述块内容完整，特别关注：

1. **提示部分**：OJ 的"提示"往往包含多种解法，不得只保留一种而丢弃其余
2. **公式推导**：OJ 原文中的公式推导、计算示例必须完整保留
3. **样例说明**：OJ 对样例的解释文字不可省略
4. **图片题目**：题目描述为图片时，必须先 OCR/转录文字，再校验文字与提示、样例的一致性

常见遗漏模式：
- 提示有多段（方法一/二/三），只复制了最后一段
- 样例解释句被跳过（如"结果为5，是因为最大覆盖区间段为：2 1 -2 3 5"）
- 自写简短描述替代了 OJ 的详细公式推导
- 代码缩进丢失：`document.body.innerText` 会丢弃空白。应从 `<pre>` 块的 `innerHTML` 提取并解码 `&lt;`/`&gt;` 实体
